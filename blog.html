What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as a tree of nodes, where each node corresponds to a part of the document (e.g., an element, attribute, or piece of text).

How do you select an element from the DOM?

To select an element from the DOM, you can use methods like `document.getElementById()` to select by ID, `document.getElementsByClassName()` to select by class name, or `document.querySelector()` and `document.querySelectorAll()` for more general and flexible selection using CSS selectors.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in which a single event listener is added to a parent element to manage events for multiple child elements. Instead of attaching separate listeners to each child element, you delegate the event to the parent. This is useful for better performance and handling dynamic content where child elements may be added or removed.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes, you can use methods like `setAttribute()` and `getAttribute()`. To change an element's styles, you can directly modify the `style` property (e.g., `element.style.color = 'red';`). You can also add or remove CSS classes using `classList.add()` or `classList.remove()`.
